# 知识小勇士 — 上线验收检查清单 (QA Checklist)

> **核心理念**：每次上线前，必须按此清单逐项验证。不是"看起来没问题"就行，而是要有**具体的对比数据**。
> 
> **使用方法**：每个检查项标注 ✅/⚠️/❌，⚠️ 和 ❌ 必须修复后才能上线。

---

## 一、三科横向一致性 (Cross-Subject Consistency)

> 这是最容易出问题的地方：数学改了一个功能，语文和英语没有同步改。

### 1.1 答题反馈特效一致性
> 每次修改任何反馈特效时，必须三科同步检查。

| 检查项 | 数学 | 语文 | 英语 | 验证标准 |
|--------|------|------|------|----------|
| 正确音效 | ✅ playCorrectSound | ✅ playCorrectSound | ✅ playCorrectSound | 三科调用同一个函数 |
| 错误音效 | ✅ playWrongSound | ✅ playWrongSound | ✅ playWrongSound | 三科调用同一个函数 |
| Confetti粒子数 | 12 (speed) / 18 (free) | 12 | 12 | ⚠️ Math Free=18，应统一为12 |
| Confetti颜色数 | 5色 | 5色 | 5色 | 颜色风格可以按科目不同 |
| Confetti大小范围 | 6+random*6 | 6+random*6 | 6+random*6 | ⚠️ Math Free=6+random*8 |
| Confetti动画时长 | 0.8s | 0.8s | 0.8s | ⚠️ Math Free=1.2s |
| Confetti起始位置 | top:40% | top:40% | top:40% | ⚠️ Math Free=top:30% |
| 反馈遮罩样式 | 全卡片 bg-white/70/80 | 全卡片 bg-white/70 | 全卡片 bg-white/70 | 统一用 bg-white/70 |
| 反馈图标大小 | w-16 (speed) / w-20 (free) | w-16 | w-16 | ⚠️ Math Free=w-20，应统一w-16 |
| 卡片缩放动画 | scale [1,1.2,1] 0.3s | scale [1,1.1,1] 0.4s | scale [1,1.1,1] 0.4s | 应统一参数 |
| 错误摇晃动画 | rotate抖动 | x轴抖动 | x轴抖动 | Math用rotate，语英用x轴，可接受 |
| Floating XP | ❌ 无 | ✅ +10 XP 橙色 | ✅ +10 XP 翡翠色 | ⚠️ Math缺少，应补齐 |
| Combo显示 | combo≥2 (free) / ❌无 (speed) | combo≥3 | combo≥3 | ⚠️ Math Speed无combo |
| Combo音效 | combo≥2 (free) / ❌无 (speed) | combo≥3 | combo≥3 | ⚠️ Math Speed无combo |
| Combo阈值 | ≥2 | ≥3 | ≥3 | ⚠️ 应统一为≥3 |

### 1.2 限时挑战行为一致性
> 每次修改Speed模式逻辑时，必须三科同步检查。

| 检查项 | 数学 | 语文 | 英语 | 验证标准 |
|--------|------|------|------|----------|
| 初始题目数 | 50 | 50 | 50 | 统一50 |
| 无限生成阈值 | 用完再生成 | ≤5题时+20 | ≤5题时+20 | Math应改为≤5提前生成 |
| 正确后延迟 | 350ms | 300ms | 300ms | ⚠️ 应统一为300ms |
| 错误后延迟 | 800ms | 800ms | 800ms | 统一800ms ✅ |
| 结束后跳转 | 800ms延迟→result | 立即显示result | 立即显示result | ⚠️ Math多了800ms |
| 紧迫色变阈值 | ≤10s | ≤10s | ≤10s | 统一 ✅ |
| Speed Badge | ✅ | ✅ | ✅ | 统一 ✅ |

### 1.3 结果页一致性
> 每次修改结果展示时，必须三科同步检查。

| 检查项 | 数学 | 语文 | 英语 | 验证标准 |

| Retry行为 | 回主页重新开始 | 原位重新生成 | 原位重新生成 | ⚠️ Math不一致 |

### 1.4 导航一致性

## 二、跨模式一致性 (Cross-Mode Consistency)

> 自由练习、限时挑战、闯关模式之间的行为是否一致。

### 2.1 题目生成源一致性

| 检查项 | 验证标准 |
|--------|----------|
| **同科目同设置下，三种模式应该生成相同类型的题目** | |
| Math: Free/Speed/Adventure 是否使用同一套题库？ | ⚠️ Speed用easy难度，Free用用户设置 |
| Chinese: Free/Speed/Adventure 是否使用同一套题库？ | ✅ 都用 generateChineseQuestions |
| English: Free/Speed/Adventure 是否使用同一套题库？ | ✅ 都用 generateEnglishQuestions |
| Math: Speed再生时是否和初始用同一源？ | ✅ 检查grade用curriculum，否则standard |

### 2.2 设置页配置是否真正生效

| 检查项 | 验证标准 |
|--------|----------|
| 设置年级后，自由练习是否使用了对应年级题库？ | 每个科目都要验证 |
| 设置年级后，限时挑战是否使用了对应年级题库？ | Math通过curriculum；语英通过config.grade |
| 设置年级后，闯关模式是否使用了对应年级题库？ | 闯关应该用关卡定义的年级（设计如此） |
| 设置学期（上/下册）是否影响题目内容？ | Math通过curriculum区分；语英暂不区分 |

### 2.3 题型选择一致性

| 检查项 | 验证标准 |
|--------|----------|
| **自由练习可选的题型，限时挑战也应该全部可选** | 上次Bug就在这里！ |
| Math: SpeedSetup题型 ⊆ Free题型？ | ⚠️ SpeedSetup有compare/equation但MathHome speed没有 |
| Chinese: Speed题型 = Free题型？ | ✅ 相同，都按年级过滤 |
| English: Speed题型 = Free题型？ | ✅ 都是全部4种 |

## 三、数据完整性 (Data Integrity)

### 3.1 题库内容质量

| 检查项 | 验证标准 |
|--------|----------|
| 每个年级×每册至少有3种以上题型 | 逐一检查math-curriculum TOPICS |
| 题型分布均匀，不应该某一种题型占比超过40% | 上次Bug：G6S2比大小占40% |
| 题目数字范围适合目标年级 | G1在10以内，G6在1000以内 |
| 干扰项（错误选项）合理，不出现重复或负数 | 适用于选择题 |
| 表达式显示正确，无"0=0=?" | 复杂题用expression字段 |

### 3.2 答案和判题正确性

| 检查项 | 验证标准 |
|--------|----------|
| 正确答案在options中能找到 | 抽样10题验证 |
| 比较题的 > 和 ≤ 逻辑正确 | 边界值测试 |
| 分数题的结果为最简分数 | |
| 方程题的解为整数 | |
| 正确率 = 答对数 / (答对数 + 答错数) | 不用题目总数作分母 |

### 3.3 状态生命周期

| 检查项 | 验证标准 |
|--------|----------|
| 计时器在组件卸载时被清理 | 三科所有timer都要检查 |
| 反馈延迟setTimeout在卸载时被清理 | ⚠️ 语英普通模式有泄漏 |
| TTS语音在返回/卸载时被停止 | |
| Audio Context在首次交互时恢复 | |
| 键盘事件监听器在卸载时移除 | |

## 六、每次发布的必做验证流程

### Step 1: 三科功能对等性 (30分钟)
```
□ 数学自由练习 → 限时挑战 → 闯关：走完全流程
□ 语文自由练习 → 限时挑战 → 闯关：走完全流程  
□ 英语自由练习 → 限时挑战 → 闯关：走完全流程
□ 对比三科：题型选择、反馈特效、结果展示 是否一致
```

### Step 2: 跨模式题库一致性 (15分钟)
```
□ 设置年级=X，学期=Y
□ 分别进入自由练习、限时挑战
□ 确认生成的题目符合该年级难度和内容
□ 切换不同年级，确认题目变化
```

### Step 3: 边界场景 (10分钟)
```
□ 限时挑战：等到时间归零，确认正常结束
□ 限时挑战：答题中按返回，确认无报错
□ 自由练习：答完所有题，确认正常结束
□ 闯关：失败后重试，确认可以重来
□ 极速连续点击，确认无重复提交
```

### Step 4: 移动端适配 (5分钟)
```
□ 375px宽度下三科Home页Tab切换正常
□ 375px宽度下答题界面不溢出
□ 弹窗（签到/设置/帮助）不被键盘遮挡
```

### Step 5: 数据验证 (10分钟)
```
□ 答完全部正确 → 结果显示100% / 满分鼓励语
□ 答完全部错误 → 结果显示0% / 鼓励性文案（非"再复习一遍"）
□ 答一部分正确 → 百分比计算正确
□ 金币、经验、星星奖励正确发放
□ 宠物经验值正确增长
```
